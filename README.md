# Punk Software (2026-07-07)

A template repository for the punk software hack night.

## What problem we're solving

What we're building today shapes how we think about working software. Projects from punk software sessions are built fast, shared openly, and iterated on without permission. Software as community artifact. This repository structure keeps code, context, prompts, and artifacts organized so every future session starts with a clear foundation.

## What's in this repo

  - `artifacts/` — Noncode artifacts: written documents, web pages, and other media content.
  - `ideas/` — Documentation and WIP ideation work.
  - `sessions/` — LLM prompts and session history.
  - `src/` — Source code.
  - `docs/` — Documentation for humans and LLMs (generated or hand-written).

## Getting started

1. Click the "Use this template" button on GitHub to create a new repo from this template.
2. Fill in `README.md` with project-specific context (including "What problem we're solving").
3. Choose your stack, create a session, and start building.
